# VideMástico — Portfólio Empresarial

## 1. Identificação

**Nome do projeto:** VideMástico  
**Tipo:** Portfólio empresarial  
**Sistema de Design utilizado:** Bootstrap 5 UI Kit  
**Ferramenta de prototipação:** Figma  
**Formato da entrega:** Protótipo navegável para Desktop  

O projeto consiste em um portfólio empresarial da VideMástico, desenvolvido por meio de diferentes frames no Figma, com foco na apresentação institucional da empresa, de seus produtos, recursos, projetos, certificações e canais de contato.

---

## 2. Resumo

O projeto foi desenvolvido como um **protótipo navegável no Figma**, voltado para Desktop e organizado em frames conectados entre si.

O portfólio apresenta a identidade visual da VideMástico e permite a navegação entre as seguintes áreas:

- Home;
- Projetos;
- Recursos e materiais;
- Sobre a empresa;
- Certificações e reconhecimentos;
- Contato.

O **Bootstrap 5 UI Kit** foi utilizado como referência para a definição visual de componentes, espaçamentos, cards, botões, formulários, menus e organização das telas.

Não há implementação em HTML, CSS ou JavaScript. A entrega é composta pelo planejamento visual e pela prototipação das telas no Figma.

---

## 3. Justificativa

O desenvolvimento do portfólio teve como objetivo apresentar a VideMástico de forma organizada, moderna e profissional.

A proposta reuniu, em um único projeto visual, informações importantes sobre a empresa, como sua atuação, seus produtos, os materiais utilizados, seus diferenciais e suas certificações.

O protótipo navegável permitiu simular a experiência de uso do portfólio e avaliar a organização das informações, a facilidade de navegação, a consistência visual e a interação entre os componentes.

O **Bootstrap 5 UI Kit** foi escolhido por oferecer uma estrutura visual conhecida, organizada e adequada para a criação de interfaces consistentes.

---

## 4. Requisitos do Portfólio

### 4.1 Requisitos gerais

O portfólio apresenta as seguintes características:

- Foi desenvolvido no Figma;
- Foi projetado para Desktop;
- Utiliza diferentes frames conectados;
- Possui navegação entre as páginas;
- Disponibiliza uma opção para voltar ao início;
- Mantém uma identidade visual consistente;
- Utiliza estilos e componentes em todo o projeto;
- Apresenta a paleta de cores utilizada;
- Aplica comportamentos intuitivos e animados em elementos importantes.

### 4.2 Navegação

O protótipo permite a navegação entre os frames principais:

- Home;
- Projetos;
- Recursos;
- Sobre a empresa;
- Certificados;
- Contato.

O menu superior foi utilizado como principal meio de navegação.

Também existe uma opção de **Voltar ao início**, permitindo que o usuário retorne facilmente para a Home a partir das demais telas.

A página atual aparece destacada no menu para facilitar a orientação do usuário.

### 4.3 HERO e banner

A página inicial possui uma seção **HERO** com banner em destaque.

O banner apresenta:

- Navegação automática entre seus elementos;
- Pelo menos um elemento navegável;
- Navegação manual;
- Botões e indicadores de posição;
- Transições suaves entre os conteúdos;
- Informações importantes da empresa em destaque.

A navegação manual é realizada por meio de:

- Setas laterais;
- Indicadores em formato de pontos;
- Botões de avançar e voltar.

### 4.4 Estilos

O projeto utiliza estilos definidos no Figma para manter a padronização entre os frames.

Foram criados estilos para:

- Títulos;
- Subtítulos;
- Textos;
- Textos auxiliares;
- Cores principais;
- Cores de apoio;
- Sombras;
- Bordas;
- Espaçamentos.

A utilização de estilos facilitou a manutenção da consistência visual em todo o projeto.

### 4.5 Componentes

Os principais elementos da interface foram criados como componentes reutilizáveis.

Componentes utilizados:

- Cabeçalho;
- Menu de navegação;
- Logotipo;
- Botões;
- Cards de produtos;
- Cards de projetos;
- Cards de certificações;
- Campos de formulário;
- Banner do HERO;
- Indicadores do carrossel;
- Rodapé;
- Botão de voltar ao início.

Também foram utilizadas variantes para representar diferentes estados dos componentes.

Exemplos:

- Botão normal;
- Botão em hover;
- Botão pressionado;
- Item de menu normal;
- Item de menu ativo;
- Card normal;
- Card em destaque.

### 4.6 Interações e animações

O protótipo possui comportamentos intuitivos e atraentes.

Foram aplicadas as seguintes interações:

- Mudança de estado dos botões ao passar o cursor;
- Destaque do item ativo no menu;
- Transições suaves entre frames;
- Animação do banner principal;
- Alteração visual dos cards durante o hover;
- Abertura de detalhes de projetos;
- Rolagem até seções específicas;
- Retorno à página inicial.

As animações são discretas e contribuem para a experiência do usuário sem prejudicar a leitura ou a navegação.

## 5. Estrutura dos Frames

### 5.1 Home

A Home apresenta:

- Cabeçalho com menu;
- HERO com banner;
- Navegação automática e manual;
- Apresentação resumida da empresa;
- Principais projetos;
- Principais produtos;
- Chamadas para outras páginas;
- Botão de contato.

### 5.2 Projetos

O frame de projetos apresenta exemplos de soluções desenvolvidas pela VideMástico.

Os projetos são apresentados em cards com:

- Imagem;
- Nome do projeto;
- Breve descrição;
- Categoria;
- Botão para visualizar mais informações.

### 5.3 Recursos e Materiais

O frame apresenta:

- Filmes plásticos;
- Embalagens;
- Sacos e sacolas;
- Produtos personalizados;
- Polietileno virgem;
- Polietileno reciclado;
- Pigmentos;
- Aditivos.

As informações são apresentadas de forma simples, evitando excesso de termos técnicos.

### 5.4 Sobre a Empresa

O frame apresenta:

- Quem somos;
- O que fazemos;
- Nosso diferencial;
- Localização da empresa;
- Valorização dos colaboradores;
- Atuação da VideMástico.

### 5.5 Certificações e Reconhecimentos

O frame apresenta:

- Referência ao Great Place to Work;
- Referência ao IFS PACsecure;
- ISO 9001;
- ISO 14001;
- ISO 45001;
- Programa 5S;
- Premiações e reconhecimentos.

Os selos e certificados estão organizados em cards e blocos visuais.

### 5.6 Contato

O frame de contato apresenta:

- Localização;
- Telefone;
- E-mail;
- Horário de atendimento;
- Formulário;
- Botão para envio;
- Opção de retorno à Home.

## 6. Sistema de Design — Bootstrap 5 UI Kit

O Bootstrap 5 UI Kit foi utilizado como referência visual para a organização dos elementos do projeto.

Foram observados conceitos como:

- Sistema de grid;
- Espaçamentos;
- Cards;
- Botões;
- Menus;
- Formulários;
- Bordas;
- Sombras;
- Hierarquia visual;
- Organização responsiva.

Como o projeto foi desenvolvido exclusivamente no Figma, o Bootstrap foi utilizado como **base de design**, e não como biblioteca de programação.

O protótipo foi desenvolvido para Desktop.

- Cabeçalho com altura consistente;

- Margens semelhantes;

- Navegação no mesmo local em todas as telas.
